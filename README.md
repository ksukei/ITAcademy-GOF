# ITAcademy-GOF
Данный репозиторий создан по программе Академии IT 2026.

Шаблон проектирования — это не библиотека и не фреймворк, а именованное решение типовой проблемы в архитектуре программы.
То есть это скорее «удачный способ организовать классы и их взаимодействие», чем конкретная реализация. Такой подход
помогает делать код более гибким, расширяемым и переиспользуемым.

Порождающие (Creational) — отвечают за создание объектов.
Структурные (Structural) — отвечают за состав и связи между классами и объектами.
Поведенческие (Behavioral) — отвечают за взаимодействие объектов и распределение ответственности между ними.

Что обычно важно понять на старте
На первых уроках обычно достаточно запомнить, что паттерны нужны, когда код начинает «ломаться» от изменений:
много if/else, жёсткая связность, сложно заменить реализацию, трудно тестировать, сложно расширять. Тогда паттерн помогает
сделать структуру более устойчивой. Но паттерны не используют «просто потому что так красиво» — это инструмент для
конкретной проблемы, а не обязательная часть каждого проекта. Последнее — это вывод из самой идеи каталога как набора
решений для повторяющихся задач, а не универсального правила на все случаи.

Порождающие — отвечают на вопрос: «как создавать объект?»

Factory Method — через фабричный метод.
Abstract Factory — семействами.
Builder — по шагам.
Singleton — один экземпляр.
Prototype — копированием.

Структурные - отвечает на вопрос: "как соединять объекты и классы"

Adapter — переходник между интерфейсами.
Bridge — отделяет абстракцию от реализации.
Composite — дерево объектов.
Decorator — добавляет поведение через обёртку.
Facade — упрощённый интерфейс к сложной системе.
Flyweight — экономия памяти.
Proxy — объект-заместитель с контролем доступа.

Поведенческие — отвечают на вопрос: «как объекты взаимодействуют?»

Strategy — выбираем алгоритм.
Observer — подписка на изменения.
Command — действие как объект.
State — поведение зависит от состояния.
Template Method — каркас алгоритма.
Chain of Responsibility — запрос идёт по цепочке.
